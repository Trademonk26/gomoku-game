## 0. 요약 (Executive Summary)

**무엇을 만드는가**: 전국 지도 위에서 데이터센터 입지 적합도를 7개 축 × 38개 지표로 점수화하고, 시도→시군구→읍면동으로 드릴다운하며, 각 지역에 대해 "왜 유망한지 / 왜 제외·주의해야 하는지 / 다음에 무엇을 확인해야 하는지"를 근거 데이터와 함께 자동 생성하는 의사결정 대시보드.

**핵심 설계 사상 4가지**:

1. **게이트 + 스코어 2단계 구조** — 실제 입지 선정은 가중합이 아니라 배제(elimination)부터 시작한다. 규제·재해로 개발 불가능한 면적을 먼저 걷어내 "개발가능면적"을 계산하고, 임계치를 넘는 지역만 랭킹에 올린다.
2. **모든 숫자에 근거·출처·신뢰도 부착** — 점수는 클릭 2번 안에 원천 데이터와 산식까지 추적 가능해야 한다. 신뢰도 등급(A/B/C)과 데이터 기준일을 지표 단위로 관리한다.
3. **프록시임을 숨기지 않는다** — 계통 접속 여유 같은 결정적 정보는 공개 데이터로 알 수 없다. 도구는 "확실한 답" 대신 **"다음에 확인할 것 리스트"를 자동 생성**하는 방향으로 설계한다. 이것이 이 도구의 정직한 가치다.
4. **시나리오별 가중치** — AI 훈련 캠퍼스(전력 최우선, 지연시간 둔감)와 클라우드 리전/코로케이션(네트워크·지연시간 민감)은 최적 입지가 다르다. 가중치 프리셋과 사용자 슬라이더로 목적별 랭킹을 제공한다.

**왜 지금인가 (한 줄 논리)**: 수도권은 계통영향평가·접속 대기열·민원으로 신규 대형 DC가 구조적으로 막히는 중이고, 정부 인센티브와 하이퍼스케일러 투자는 비수도권으로 이동하고 있다. "전력 접속이 가능한 대규모 비수도권 부지"가 희소자산이 되는 구간이며, 공개 데이터만으로도 후보를 남보다 먼저 좁힐 수 있다.

## 1. 배경: 시장·정책 컨텍스트 (2026-07 기준, 검증됨)

스크리닝 로직 전체가 아래 구조적 비대칭 위에 서 있다. 각 항목은 웹 검증을 거쳤고, 변동 가능성이 있는 것은 `[요확인]`으로 표시했다.

### 1.1 수도권 억제 (Push)

- **전력계통영향평가**: 분산에너지 활성화 특별법(2023.6 제정, 2024.6 시행)에 따라 대규모 신규 전력수요 시설(데이터센터 등)은 계통영향평가를 통과해야 전력을 공급받는다. 수도권 등 계통 포화지역의 신규 대형 DC를 억제하는 것이 명시적 취지다. "전력 쪼개기"(수전 용량 분할로 평가 회피)도 제도적으로 차단되어 있다(기후에너지환경부 설명자료).
- **접속 대기열 포화**: 2029년까지 신규 DC 접속 신청 601건·49.4GW 중 약 86%가 수도권 집중(산업부 2023.3 실태조사). 실제 착공으로 이어지는 비율은 낮아 대기열 자체가 허수로 오염 — 신청해도 언제 연결될지 모르는 상태.
- **민원에 의한 인허가 실패가 통계적으로 유의미**: 2024~2025년 수도권 DC 인허가 33건 중 17건(52%)이 주민 반대로 지연·무산. 김포 구래동(디지털리얼티) 무기한 연기, 안양 호계동(효성) 취소, 김포·고양의 착공신고 반려→행정심판 사례. 핵심 갈등 요인은 **특고압 지중선 전자파(학교·주거 인근)** 와 소음, 지역경제 기여 의문.

### 1.2 비수도권 유인 (Pull)

- **한전 특례**: 비수도권 신설 DC에 22.9kV 접속 시 시설부담금 50% 할인, 154kV 접속 시 예비전력 요금 면제(2023.6~2026.5 한시). `[요확인: 2026.5 만료됨 — 연장/개편 여부가 스크리닝 가중치에 직접 영향]`
- **분산에너지특화지역**: 특구 입주 DC는 계통영향평가 비기술 항목 면제 방침(2025.4 발표). 특구 내 직접 전력거래(PPA) 허용. `[요확인: 특구 지정 현황 목록]`
- **지역별 차등 전기요금제**: 분산법에 근거 조항 존재. 시행되면 비수도권 전력비 우위가 구조화됨. `[요확인: 시행 시기·요율]`
- **하이퍼스케일러의 비수도권 베팅이 이미 시작**: SK-AWS 울산 미포국가산단 AI DC(103MW, 2025.8 기공, 1단계 2027.11 가동 목표), SKT 영남권 2GW AIDC 구상 및 2035년까지 국내 15GW 인프라 계획 발표(2026.7 보도). 광주 NHN 국가AI데이터센터, 네이버 각 세종(2023 가동) 등 선례 축적.

### 1.3 기술 트렌드가 입지 조건을 바꾸는 중

- AI 훈련 캠퍼스는 50~300MW+ 급으로 대형화 — 22.9kV 배전 접속(통상 ~10MW급)으로는 불가, **154kV 전용선로 이상이 전제**. 따라서 "변전소 근접성·여유"가 부지 가치의 제1 결정 변수. `[요확인: 한전 송·배전용 전기설비 이용규정상 전압별 계약전력 기준]`
- 변전소 신설 리드타임 5~7년, 송전선로 신설은 10년+(입지 갈등 시 무기한) — **기존 계통 여유를 선점하는 쪽이 구조적으로 유리**.
- 액체냉각(DLC) 확산으로 물 소비는 감소 추세지만, 냉각탑 보충수·비상 냉각용 안정적 용수는 여전히 필수. 냉량 기후(낮은 냉방도일)는 PUE에 직접 기여.
- AI 훈련 워크로드는 사용자 지연시간에 둔감(배치성) — 통신축 가중치를 낮출 수 있어 **입지 자유도가 과거 DC보다 크다**. 반면 클라우드 리전·서빙은 여전히 수도권 왕복 수 ms 이내가 유리.

## 2. 제품 정의

### 2.1 사용자와 핵심 질문

| 페르소나 | 핵심 질문 | 도구가 주는 답 |
|---|---|---|

| 투자 심사역 | "이 지역이 3위인 근거는 무엇이고 얼마나 믿을 만한가" | 축별 기여도 분해, 근거 팝오버, 순위 안정성 |

### 2.2 하는 것 / 하지 않는 것

| 하는 것 | 하지 않는 것 |
|---|---|
| 전국 단위 상대 비교·랭킹 | 특정 필지의 매수 판단 |
| 근거·출처가 달린 추천/주의/제외 사유 | 계통 접속 보장, 인허가 결과 예측 |
| 실사 전 확인 체크리스트 생성 | 지가 전망, 수익률 계산 |
| 시점 스냅샷 비교(개선/악화 추세) | 토지 소유자 개인정보 취급 |

### 2.3 핵심 사용자 여정 (3단계)

**탐색**(전국 코로플레스에서 상위 지역 파악) → **비교**(후보 2~4개 레이더·지표 테이블 대조) → **검증 준비**(지역 원페이저 리포트 + 확인 체크리스트 내보내기 → 실사·기관 문의).

### 2.4 성공 기준 (제품 KPI)

- 229개 시군구에서 답사 후보 10개 이하로 좁히는 데 **30분 이내**.
- 화면의 모든 점수에서 원천 출처·산식·기준일까지 **2클릭 이내** 도달.
- 분기 데이터 갱신: 파이프라인 자동 실행 1시간 이내 + 수동 큐레이션(조례·뉴스) 4시간 이내.
- 가중치를 바꿔도 흔들리지 않는 지역(순위 안정성 상위)을 별도 식별 가능.

## 3. 입지 평가 프레임워크 (스코어링 모델)

### 3.1 2단계 구조: 게이트(배제) → 스코어(랭킹)

**단계 1 — 개발가능면적 게이트.** 아래 레이어를 지역 면적에서 차감해 `개발가능면적(developable_area)`을 계산한다. 시군구 레벨에서 이 값이 임계치(기본 30ha, 시나리오별 조정) 미만이면 랭킹에서 제외하고 사유를 표시한다. 지역 전체를 이분법으로 죽이지 않고 **"가용 면적"이라는 연속량으로 반영**하는 것이 핵심 — 규제가 많아도 넓은 지역은 살아남는다.

차감 레이어(합집합): 개발제한구역 · 상수원보호구역 · 문화재보호구역 · 자연공원 · 생태자연도 1등급 · 군사기지 통제/제한보호구역 · 농업진흥구역 · 보전산지 · 100년 빈도 침수예상지 · 경사 8° 초과 또는 표고 300m 초과 지형.

**단계 2 — 7축 가중합 스코어.** 게이트를 통과한 지역만 0~100점으로 랭킹.

```
총점(r) = clamp( Σ_a  w_a × A_a(r)  +  페널티(r), 0, 100 )

  A_a(r)     = Σ_i  v_ai × pct(x_i(r))      # 축 내 지표 가중평균
  pct(·)     = 전국 백분위 순위(0~100), 방향 정렬 후  # 이상치에 강건
  결측 처리   = 해당 지표 가중치를 축 내 재분배 + 커버리지(%) 별도 표시 (0점 처리 금지)

  페널티: 계통 포화지역(수도권 등 계통영향평가 대상권) −15
          지자체 DC 제한 조례 존재 −10
          계통 고립(제주) −10

  등급: S ≥ 85 / A ≥ 75 / B ≥ 65 / C ≥ 50 / D < 50   # UI는 등급 우선 표기(과잉 정밀도 방지)
```

### 3.4 신뢰도·불확실성의 1급 시민화

- **커버리지 뱃지**: 지역 점수 옆에 "데이터 커버리지 82% · 신뢰도 C 지표 기여 18%"를 상시 표기.
- **순위 안정성**: 가중치를 ±20% 범위에서 200회 무작위 섭동(Dirichlet)해 순위 분포를 계산, IQR로 5단계 표시(●●●●●). 가중치를 조금만 바꿔도 뒤집히는 랭킹은 노이즈임을 사용자에게 정직하게 보여준다.
- **읍면동 상속 표시**: 시도·시군구 단위로만 존재하는 지표(자급률 등)는 하위 지역에 상속하되 "상위지역 값" 뱃지를 붙인다.
- **스냅샷 버저닝**: `scores_2026Q3` 형식으로 분기별 동결, 시점 간 순위 변동(개선/악화)을 추세 화살표로 표시.

### 4.3 파이프라인 (배치, 분기 1회 + 수동 트리거)

```
ingest(소스별 모듈, Python)                    # 원천 → data/raw/{source}/{date}/
  → staging(GeoParquet 정규화)                 # 좌표계·행정코드(법정동코드 10자리) 통일
  → indicators(GeoPandas/DuckDB Spatial)       # 38개 지표 계산, 지표별 단위테스트
  → scoring(정규화·가중합·게이트·민감도)          # py와 웹의 scoring 로직은 동일 fixture로 계약 테스트
  → publish                                    # outputs/scores_YYYYQ.json, boundaries/, evidence_YYYYQ.json
```

- 오케스트레이션: GitHub Actions cron(분기) + `make all` 재현 가능 로컬 실행.
- 행정구역 개편(통합·분동) 대응: 스냅샷마다 행정코드 매핑 테이블 동결.

### 4.4 데이터 모델 (핵심 엔티티)

```

sources      (id PK, name, url, license, last_checked)

Phase 1~2는 이 모델을 정적 JSON으로 직렬화해 서빙(DB 서버 불필요). Phase 3에서 PostGIS로 이관.

## 5. 설명가능성: 추천·제외 사유와 체크리스트 자동 생성

점수는 신뢰를 만들지 못한다. 사유가 만든다. **LLM이 사실을 생성하는 방식이 아니라, 규칙 기반 템플릿에 실데이터를 채우는 방식**으로 구현한다(환각 원천 차단).

| 종류 | 트리거 규칙 | 템플릿 예시 |
|---|---|---|
| 추천 | 축점수 상위 15% 또는 개별 지표 상위 10% | "「154kV 변전소 근접성」 전국 상위 8% (최근접 4.2km) — 대용량 수전 인프라 기확보 지역" |
| 주의 | 축점수 하위 20% 또는 지표 하위 15% | "「서울 IX 지연시간」 하위 12% (왕복 ~5.1ms 추정) — 저지연 서비스용으로는 부적합" |
| 제외 | 게이트 트리거 | "개발가능면적 18ha < 임계 30ha (군사시설보호구역 61% 중첩)" |
| 확인 필요 | ① 신뢰도 C 지표가 총점 기여 상위 3위 내 ② 게이트 경계값 ±10% ③ 페널티 인접 조건 | "「접속 여유 프록시」는 재생에너지 연계 기준 — **한전 ○○변전소 접속검토 신청으로 확인 필수**" |

- **기여도 분해(워터폴)**: 전국 평균 대비 이 지역 총점 차이를 축별 기여로 분해해 시각화 — "왜 3위인가"에 대한 정량 답변.
- '확인 필요' 목록은 그대로 **실사 체크리스트(부록 A)의 지역 맞춤판**이 되어 PDF 리포트에 포함된다. 이 도구의 최종 출력물은 점수가 아니라 "다음 행동 목록"이다.

### 6.4 인터랙션 원칙

- **가중치 변경 → 즉시 재계산**: 점수 합성이 클라이언트에서 일어나므로(사전계산된 지표 백분위 × 가중치) 300ms 디바운스로 전 지역 리랭킹.
- 코로플레스 색: 순차 팔레트(colorblind-safe), 게이트 제외 지역은 회색 빗금, 신뢰도 낮은 지역은 점 패턴 오버레이.
- 호버 툴팁: 총점·등급 + 최상위 사유 1줄. 클릭: 우측 패널. 더블클릭: 드릴다운.
- 비교 뷰: 레이더 중첩 + 지표 테이블(열=지역), 최대값 셀 하이라이트, 차이 큰 지표 자동 정렬.
- 다크/라이트 모드, 키보드 내비게이션(지역 검색→Enter→패널 포커스).

### 7.1 스택 선정과 근거

| 계층 | 선택 | 근거 |
|---|---|---|
| 지도 렌더링 | **MapLibre GL JS** + PMTiles | 오픈소스, 벡터타일로 읍면동 3,500 폴리곤 60fps. 구글맵은 국내 정밀지도 제약·비용으로 배제 |
| 배경 지도 | V-World 타일(무료 키) 또는 Protomaps 셀프호스팅 | 국내 행정 표준 + 비용 0 |
| 프론트엔드 | React 18 + TypeScript + Vite | 표준 선택, 채용·유지보수 용이 |
| 상태 | Zustand + URL 상태(가중치·선택 지역 공유) | 서버 없는 협업 공유 |
| 차트 | ECharts (레이더·게이지·워터폴) | 필요 차트 유형 기본 제공 |
| 점수 계산 | 클라이언트 TS (`scoring.ts`) | 서버 불필요·가중치 실시간. 파이프라인 `scoring.py`와 동일 fixture 계약 테스트 |
| 배포 | Vercel 또는 GitHub Pages (정적) | Phase 1~2 서버 비용 0원 |
| 파이프라인 | Python 3.12, GeoPandas, DuckDB Spatial, GitHub Actions | GIS 오버레이·대량 조인에 충분, 무료 CI |
| Phase 3 백엔드 | FastAPI + PostGIS | 필지 탐색·알림은 서버 필요 시점에만 도입 |

**데이터 예산 검증**: 시군구 229 × 38지표 ≈ 수백 KB(gzip 후 ~100KB) — 전 지표를 초기 로드해도 무리 없음. 읍면동은 PMTiles 속성으로 내장해 타일 단위 로드.

### 7.2 저장소 구조

```
Map/
├── BLUEPRINT.md              # 본 문서
├── pipeline/                 # Python ETL
│   ├── sources/              #   소스별 ingest 모듈 (kepco.py, vworld.py, ...)
│   ├── indicators/           #   지표 계산 (축별 모듈, 단위테스트 동반)
│   ├── scoring/              #   정규화·가중합·게이트·민감도 (scoring.py)
│   ├── curation/             #   수동 큐레이션 입력 (조례, 유치의지 루브릭 — YAML)
│   └── outputs/              #   scores_YYYYQ.json, evidence_YYYYQ.json, boundaries/
├── web/                      # React SPA
│   └── src/
│       ├── components/       #   Map, RankingTable, RegionDetail, WeightPanel, Compare
│       └── lib/scoring.ts    #   scoring.py와 계약 테스트로 동기화

### 7.3 품질 기준

- 지표 모듈별 단위테스트 + 점수 회귀 스냅샷 테스트(스냅샷 간 의도치 않은 순위 급변 감지).
- `scoring.py ≡ scoring.ts` 계약 테스트(동일 fixture, 허용오차 1e-9).
- LCP < 2.5s, 지도 인터랙션 60fps, 모든 색상 WCAG AA.

## 10. 리스크·한계·거버넌스 (정직한 고지)

1. **계통 여유 데이터의 근본 한계**: 공개된 접속가능용량은 재생에너지 연계 기준으로, DC 수전 여유와 다르다. 이 축의 모든 지표는 프록시이며, 확정은 한전 접속검토(유상) 뿐이다. → UI에 상시 명시, '확인 필요' 자동 생성.
2. **제도 변동성**: 한전 특례(2026.5 한시)의 연장 여부, 차등요금제 시행, 분산특구 지정이 가중치 구조를 바꾼다. → 분기 스냅샷 + 정책 변경 시 수시 스냅샷.
3. **수용성 정량화의 위험**: 민원 리스크는 본질적으로 정성적이다. A3(뉴스 건수)는 노이즈가 크므로 근거 기사 링크 없이는 점수에 반영하지 않는다.
4. **지도·데이터 라이선스**: V-World 이용약관(타일 캐싱), 공공누리 출처표시, TeleGeography 재배포 금지 준수. 육양국 위치는 공개 문헌 기반 자체 정리본만 사용.
5. **다루지 않는 것**: 토지 소유자 개인정보, 실거래 호가, 특정 필지 매수 권유 — 법적 리스크 원천 차단.
6. **결론의 성격**: 이 도구의 출력은 "어디를 먼저 검증할 것인가"이지 "어디를 살 것인가"가 아니다.
