## Workflow Rules

### Git Commit & Push

- 모든 변경사항은 작업 완료 후 반드시 git commit & push 한다.
- 커밋 메시지는 **한국어**로 작성하며, 나중에 어떤 변경을 했는지 알아볼 수 있도록 명확하고 구체적으로 작성한다.
- 커밋 메시지 형식:
  - 첫 줄: 변경 요약 (예: `사용자 인증 기능 추가`, `로그인 버그 수정`)
  - 본문(필요시): 변경 이유, 주요 변경 내용 상세 설명
- 하나의 작업 단위(기능 추가, 버그 수정 등)마다 커밋을 나눈다.
- push 대상 remote: `origin`
